# 2026-06-30

## status.json / result.json now bind through df/dd

terminus's own review artifacts moved from `encoding/json` to `df/dd`, to satisfy
the `df-binding` canon quality terminus itself enforces (caught by a terminus
self-review). `monitor.go` (status.json) and `broker/review.go` (result.json, via
`writeJSONAtomic` + `CollectReview`) now use `dd.UnbindJSON`/`dd.BindJSON`. The
non-obvious, load-bearing points:

- **No `dd` struct tags were added, deliberately.** dd keys off `dd:"..."` tags,
  but its no-tag fallback is `toSnakeCase(fieldName)`, which reproduces the
  existing `json:"..."` keys *exactly* (including `ReviewID` -> `review_id`).
  Verified before migrating. The persisted structs keep only their `json` tags;
  don't add `dd` tags expecting them to change anything.
- **dd corrupts `json.RawMessage`** — generic unbind renders the underlying
  `[]byte` as a JSON number array. `reviewResultFile.Raw` (the reviewer's raw
  output) therefore round-trips through `rawMessageConverter` + `ddJSONOpts`
  (broker/review.go), which passes it as embedded JSON. monitor needs no converter
  (ReviewStatus has no RawMessage). Any new persisted struct with a RawMessage
  field must pass `ddJSONOpts`.
- **The on-disk format changed on purpose:** keys are now alphabetically sorted
  (dd -> map -> MarshalIndent), and `omitempty` is NOT honored, so empty fields
  now appear. Accepted — these files are internal/ephemeral, old review dirs still
  bind (keys match), and the MCP/CLI wire format is unaffected (serialized
  separately by the SDK/printer, not from these files). Don't "fix" the verbosity.
- **result.json's stored `Raw` is re-serialized (key-sorted)** by the converter —
  semantically identical, and the verbatim reviewer output is preserved untouched
  in `_findings.md` (written via `record`, not dd). The in-memory collect path
  returns the original Raw; only collect-from-disk re-serializes.
- **`classifiedSection` (review.go) stays on `encoding/json` deliberately** — it
  renders a JSON block into the `_findings.md` record for human display, not data
  persistence. That's the boundary; the self-review didn't flag it.
- Round-trip is guarded by `broker.TestCollectFromDiskRoundTrip` (write by one
  broker, collect-from-disk by a second; verdict/findings/Raw preserved) and
  `monitor.TestStatusRoundTrip` (monitor was at 0% coverage before).
